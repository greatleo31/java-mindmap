# 回滚到前一个提交状态操作指南

本文说明如何在 Antigravity 或其他 IDE 中，把项目回滚到前一个 Git 提交状态。

## 先区分三种情况

### 1. 只撤销未提交改动

适合场景：

- 文件改坏了，但还没有 commit。
- 只想恢复到当前最新提交的干净状态。

命令：

```powershell
git restore .
```

如果只撤销某一个文件：

```powershell
git restore src/com/hyk/mindmap/ui/MapTab.java
```

### 2. 回滚到前一个提交

适合场景：

- 当前最新 commit 不想要了。
- 想让整个项目回到上一个 commit 的状态。

命令：

```powershell
git reset --hard HEAD~1
```

注意：这个命令会丢弃当前未提交改动，也会把最新提交从当前分支移走。

### 3. 已经 push 后撤销提交

适合场景：

- 最新提交已经推送到远端。
- 不想破坏远端提交历史。

推荐命令：

```powershell
git revert HEAD
```

这会新增一个反向提交，用来抵消上一个提交的影响。

## Antigravity 中一步步操作

### 撤销未提交改动

1. 打开 Antigravity。
2. 打开当前项目。
3. 点击左侧的 Source Control / 源代码管理。
4. 在 Changes / 更改列表中找到被修改的文件。
5. 右键该文件。
6. 选择 Discard Changes / 放弃更改。
7. 在终端中检查状态：

```powershell
git status --short
```

如果没有输出，说明工作区已经干净。

### 回滚到前一个提交

1. 打开 Antigravity 的终端。
2. 先查看当前状态：

```powershell
git status --short
```

3. 查看最近提交：

```powershell
git log --oneline -5
```

4. 如果确认当前未提交改动和最新提交都不要了，执行：

```powershell
git reset --hard HEAD~1
```

5. 再次确认：

```powershell
git status --short
git log --oneline -3
```

如果 `git status --short` 没有输出，说明工作区干净。

## IDE 通用操作

大部分 IDE 的 Git 操作逻辑类似。

### 撤销未提交改动

1. 打开 Git / Version Control / Source Control 面板。
2. 找到 Local Changes / Changes。
3. 选择要撤销的文件。
4. 右键选择 Discard / Revert / Rollback。
5. 确认后，IDE 会把文件恢复到当前最新提交状态。

### 回滚到指定提交

1. 打开 Git Log / History。
2. 找到要回到的提交。
3. 右键该提交。
4. 选择 Reset Current Branch to Here / Reset HEAD to Here。
5. 选择 Reset 类型：

| 类型 | 效果 |
| --- | --- |
| Soft | 回到目标提交，但保留后续改动到暂存区 |
| Mixed | 回到目标提交，但保留后续改动到工作区 |
| Hard | 完全回到目标提交，丢弃后续提交和本地改动 |

如果目标是“彻底回到前一个提交的状态”，选择 Hard。

## 推荐安全做法

回滚前先看状态：

```powershell
git status --short
git log --oneline -5
```

如果担心丢失当前改动，先保存补丁：

```powershell
git diff > before-reset.patch
```

然后再回滚：

```powershell
git reset --hard HEAD~1
```

以后如果想恢复刚才保存的改动，可以尝试：

```powershell
git apply before-reset.patch
```

## 当前项目示例

如果当前最新提交是：

```text
051a04a 添加czj部分注释
b87df58 最终修改2
```

执行：

```powershell
git reset --hard HEAD~1
```

之后当前分支会回到：

```text
b87df58 最终修改2
```

## 风险提醒

- `git restore` 只撤销未提交改动。
- `git reset --hard HEAD~1` 会丢弃未提交改动，并移除当前最新提交。
- 已经 push 到远端的提交，不建议直接 `reset --hard` 后强推。
- 已经 push 的情况优先用 `git revert HEAD`。
- 不确定时，先执行 `git diff > before-reset.patch` 留一个恢复补丁。
